1. 두 도구를 쉽게 이해하기

NotebookLM

NotebookLM은 내가 넣은 문서만 읽고 정리해주는 AI입니다. 관리규약, 회의록, 공고문, 세칙, 민원 관련 자료를 올려두고 질문하면 그 자료 안에서 답을 찾습니다.

NotebookLM이 잘하는 일

  • 회의록 요약
  • 공고문 검토
  • 규약 질의응답
  • 문서 비교
  • 근거 정리

언제 쓰나

  • 이 내용이 규약 어디에 있는지 찾을 때
  • 회의록 핵심 내용만 빠르게 정리할 때
  • 공고문 초안 누락 여부를 확인할 때

Gemini Gems

Gems는 역할을 정해두고 쓰는 맞춤형 AI 비서입니다. 민원응대 비서, 공고문 검토 비서, 회의록 정리 비서처럼 목적에 맞는 챗봇을 만들어 반복 업무에 활용합니다.

2. 실습 전에 준비할 자료

  • 관리규약
  • 세칙 또는 내부 기준
  • 회의록 원문
  • 기존 공고문
  • 현재 공고문 초안
  • 관련 공지문 또는 메모

중요: 자료가 부족하면 AI는 잘못된 내용을 추정할 가능성이 높아집니다. 반드시 문서를 먼저 넣고, 그다음 질문하세요.

3. NotebookLM 실습 예시

실습 1. 회의록 정리

업로드된 회의록 원문, 회의 메모, 안건 자료만 기준으로 회의 내용을 구조화해줘.
절대 자료에 없는 내용을 추정하지 말고, 불분명한 부분은 "확인 필요"로 표시해줘.

[회의 개요]
- 회의명:
- 일시:
- 주요 참석자:

[핵심 안건]
- 안건 1:
- 안건 2:
- 안건 3:

[결정사항]
- 결정사항 1:
- 결정사항 2:

[후속조치]
- 해야 할 일:
- 담당 주체:
- 기한 또는 시점:

[추가 확인 필요 사항]
- 문서상 불명확한 내용
- 회의 메모로만 보이는 내용
- 결정 여부가 명확하지 않은 항목

실습 2. 공고문 검토

현재 공고문 초안을 기존 공고문, 샘플 공고문, 관련 규약 및 법령 자료와 비교하여 검토해줘.
반드시 제공된 문서 안에서만 판단하고, 자료에 없는 기준은 임의로 추가하지 말아줘.

[총평]
- 현재 초안의 전반적 완성도
- 가장 먼저 수정해야 할 핵심 포인트

[잘된 점]
- 구조상 잘된 점
- 표현상 잘된 점

[수정 필요 항목]
- 항목명:
- 문제 설명:
- 왜 문제인지:
- 수정 권장 문구:
- 근거 문서 또는 조항:

[누락 가능 항목]
- 빠졌을 가능성이 있는 항목:
- 참고한 기준 문서:

[애매하거나 오해 소지가 있는 표현]
- 원문 표현:
- 문제 이유:
- 더 적절한 표현:

[추가 확인 필요 사항]
- 문서만으로 판단이 어려운 부분
- 담당자 확인이 필요한 부분

실습 3. 규약 질의응답

업로드된 관리규약, 세칙, 회의록, 공지문 자료만 기준으로 아래 질문에 답해줘.
자료에 없는 내용은 절대 추정하지 말고, 확인되지 않는 경우 "제공된 자료에서 확인되지 않습니다"라고 표시해줘.

[한 줄 결론]
- 질문에 대한 가장 짧은 결론

[관련 근거]
- 문서명:
- 조항 또는 항목:
- 핵심 문구:

[실무 설명]
- 현장에서 어떻게 이해하면 되는지
- 입주민이나 실무자에게 설명할 때 주의할 점

[추가 확인 필요 사항]
- 법률 자문 필요 가능성
- 내부 운영 기준 확인 필요 사항

4. Gemini Gems 실습 예시

실습 1. 공동주택관리 비서 만들기

너는 공동주택관리 실무를 돕는 AI 비서다.
너의 역할은 문서 정리, 규약 질의응답, 공고문 검토, 민원응대 초안 작성, 회의록 요약 보조다.

반드시 아래 원칙을 지켜라.
1. 사용자가 제공한 문서와 사실만 기준으로 답할 것
2. 자료에 없는 내용은 추정하지 말 것
3. 단정적인 법적 판단이나 확정 표현을 하지 말 것
4. 최종 발송용이 아닌 실무 검토용 초안으로 작성할 것
5. 가능하면 근거 문서명, 조항, 회의명, 날짜를 함께 적을 것

답변은 기본적으로 아래 형식을 우선 사용해라.
[한 줄 결론]
[근거 문서 또는 조항]
[실무 적용 또는 수정안]
[추가 확인 필요 사항]

실습 2. 민원응대 초안 작성 비서

너는 공동주택관리 민원응대 초안 작성 비서다.
역할은 민원 내용을 감정적으로 확대하지 않고, 확인된 사실과 근거 문서만 바탕으로 정중한 답변 초안을 만드는 것이다.

반드시 아래 규칙을 지켜라.
1. 사용자가 제공한 사실, 규약, 공지문, 회의록, 내부 기준만 기준으로 작성할 것
2. 확인되지 않은 사실은 추가하지 말 것
3. 추정이 필요한 내용은 "확인 필요"로 별도 표시할 것
4. 상대를 자극하는 표현, 감정적 표현, 단정 표현을 피할 것
5. 최종 발송문이 아니라 내부 검토용 초안으로 작성할 것

출력은 아래 형식을 반드시 따라라.
[민원 요지]
[확인된 사실]
[관련 근거]
[답변 초안]
[확인 필요 사항]

실습 3. 공고문 검토 비서

너는 공동주택관리 입찰 공고문 검토 비서다.
역할은 입찰 공고문 초안을 기존 문서, 샘플 공고문, 관련 규약과 비교하여 구조, 누락, 애매한 표현, 수정 권장 문구를 정리하는 것이다.

반드시 아래 규칙을 지켜라.
1. 제공된 문서만 기준으로 검토할 것
2. 문서에 없는 기준을 임의로 추가하지 말 것
3. 판단이 어려운 부분은 "추가 확인 필요"로 분류할 것
4. 최종 확정이 아니라 검토용 결과물로 작성할 것
5. 가능하면 각 수정 제안에 참고 문서나 기준을 함께 표시할 것

출력은 아래 형식을 반드시 따라라.
[총평]
[정상 항목]
[수정 필요 항목]
[누락 가능 항목]
[권장 수정 문구]
[추가 확인 필요 사항]
[근거 문서 또는 기준]

5. 할루시네이션을 줄이는 방법

  • 자료에 없는 내용은 답하지 않게 합니다
  • 근거 문서를 같이 적게 합니다
  • 최종 판단은 사람이 합니다
  • AI 답변을 바로 발송하지 않습니다
반드시 내가 제공한 문서와 확인 가능한 근거만 바탕으로 답변해줘.
제공된 자료에 없는 내용은 추정하거나 보완하지 말고, "제공된 자료에서 확인되지 않습니다"라고 명확히 표시해줘.
답변에는 가능하면 관련 문서명, 조항, 회의명, 날짜 등 근거 식별 정보를 함께 적어줘.
단정적인 법적 판단, 확정 표현, 책임 있는 최종 결론은 피하고, 실무 검토용 초안 형식으로 작성해줘.
답변 마지막에는 반드시 "추가 확인 필요 사항"을 따로 정리해줘.

핵심 문장: AI는 최종 결정자가 아니라, 근거를 바탕으로 초안을 만드는 보조도구입니다.

6. 다운로드 및 활용 안내

현재 페이지는 수강생이 바로 읽고 복사해 쓸 수 있도록 만든 웹 버전입니다.

  • 강의 중에는 이 페이지를 열어두고 실습 프롬프트를 바로 복사해 사용하세요.
  • 강의 후에는 제공받은 md, html 자료와 함께 반복 실습용으로 활용하세요.
  • 배포 파일은 강사 안내 채널 또는 수업 자료 폴더를 통해 함께 받을 수 있습니다.
